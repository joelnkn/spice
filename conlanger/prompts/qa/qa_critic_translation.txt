SPECIAL CONSISTENCY RULES:

1. **Missing new_words entries:**
   - If the translation uses a word that:
       • is NOT in the lexicon,  
       • is NOT analyzable as base + valid affixes, and  
       • is NOT in the “new_words” list,

2. **Lexicon conflicts:**
   - For each conflict in LEXICON CONFLICTS:
       • Add a **priority 1 "consistency_error"**.  
       • The correction must rename or adjust POS/translation.
       • If the existing translation aligns with the new one, just request that this term be removed from new words.

   - Only affixes in the inventory may be used.
   - Must be used with correct function and order.
   - Affixes cannot be treated as new words.

4. **Derived forms:**
   - Allowed only if the derivation (POS→POS) is legal.
   - If illegal → "affix_error"
   - DO NOT treat them as new words.

5. **Necessity of new words:**
   For each invented base lexeme:
   - If the meaning could have been expressed using a different existing word or paraphrase, add a "lexical_error"**.
   - If the word resembles English or violates orthography, report a priority 1 "orthography_violation".

6. **Gloss correctness:**
   - Glosses must match affix boundaries and true functions.
   - Glossing mistakes are `"glossing_error"`.

NEVER include minor errors. DO NOT NITPICK! Be concise and clear with fixes.

Output rules:
- PRIORITY MUST be 1 or 2 ONLY. Never 0 or "none".
- If something is not an issue, do NOT include it.
- Output ONLY the JSON.
